# Built-In CSS Support

Since MWAP is built around [Webpack](https://webpack.js.org) we include a few commonly used patterns out of the box such as CSS and [CSS Module](https://github.com/css-modules/css-modules#readme) support.

CSS is by default processed by [PostCSS](https://www.npmjs.com/package/postcss) with [Autoprefixer](https://www.npmjs.com/package/autoprefixer) and minimized using [esbuild](https://esbuild.github.io/).

## Global CSS

If you wish to provide global styles, it's best to define a `styles.css` and import it in your custom `app.tsx` component.

## CSS Modules

Consider writing your component styles as [CSS Modules](https://github.com/css-modules/css-modules#readme), there are many benefits such as no more naming conflictions, explicit dependencies, no global scope and much more.

MWAP automatically compiles css files that end in `.module.css` as CSS Modules. All you have to do is import these files into your components!
